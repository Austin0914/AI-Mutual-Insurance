## 撰寫規範

每份 spec 建議包含：

1. **對應白皮書章節**（連結，不重貼大段內容）
2. **狀態變數與參數**（名稱、型別、單位、範圍、預設示意值）
3. **規則 / 公式**（精確到可寫成程式碼）
4. **邊界條件與不變量（invariants）**
5. **待解問題**（標記未定案處，連回白皮書 §9）

> 參數的「示意數字」必須明確標註為待校準，避免被當成最終值寫死。

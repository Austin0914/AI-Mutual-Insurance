# 規格：分層償付（Tranche）

> 對應白皮書 [§6.3 分層償付結構](../BluePrint/AI共險基金白皮書.md) 與 §7.3。
> 狀態：草稿。所有數字為示意值，**待校準**。

## 目的

將每筆損失的承擔切成三層，依序由下往上填補，以**設計保證償付能力**（池子負債永遠有界）。

## 參數

| 參數     | 意義                                 | 型別 / 單位          | 示意值（待校準） |
| -------- | ------------------------------------ | -------------------- | ---------------- |
| `d`      | 使用者自負額比例                     | 0–1（基點 bps 表示） | 0.20             |
| `k`      | 共同池賠付上限倍率（coverage ratio） | 倍數                 | 10               |
| `C_user` | 該使用者累積貢獻額                   | 代幣最小單位         | —                |
| `L`      | 單筆損失金額（鑑定認定）             | 代幣最小單位         | —                |

## 規則（依序填補）

對單筆損失 `L`：

1. **第一層 — 自負額**
   使用者永遠先自負 `D = d × L`。
   池子可賠基準額 `L' = L − D`。

2. **第二層 — 共同池（設上限）**
   共同池對此索賠最多賠付 `cap = k × C_user`。
   共同池實付 `P_pool = min(L', cap, 池餘額相關約束)`。

3. **第三層 — 巨災層**
   超過共同池能力、且屬系統性 / 模型級缺陷者，由提供商資深質押優先承擔；再超過則回到外部再保險或法律途徑。

## 不變量（invariants）

- `D ≥ 0`，且使用者每筆損失都有非零自負（除非 `L = 0`）。
- 共同池對任一索賠的賠付 `≤ k × C_user`，使總負債有界。
- 池子賠付不得使餘額為負；不足時採 **pro-rata haircut**（按比例打折），而非破產。

## 待解問題

- 巨災層與共同池的界線判定（何謂「系統性 / 模型級」）需鑑定流程支援，連結白皮書 §9。
- `d`、`k` 的校準依賴 [dynamic-rate.md](dynamic-rate.md) 的經驗資料。
